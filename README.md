# The Last 24 — `thelast24.in`

**Everything that mattered in India, in five minutes.**

An automated India news brief. A scheduled pipeline collects headlines from
verified publishers, writes editorial articles with Claude, sources relevant
images, and publishes a static site to GitHub Pages — refreshed every 6 hours.
It also produces a UPSC-style Current Affairs section, a FIFA World Cup 2026
live module, a tweet queue, daily Instagram carousels, LinkedIn document
carousels, and pushes everything to social via Buffer.

- **Live:** https://thelast24.in
- **Hosting:** GitHub Pages (static) + GitHub Actions (scheduled builds)
- **Cost model:** free hosting; only the Anthropic API + (optional) image APIs
  cost money. Tuned to roughly $20/month at the 6-hour cadence.
- **Model:** `claude-sonnet-4-6`
- **Analytics:** Google Analytics `G-B1R3X3GKJ3` (Consent Mode v2, denied by default)
- **Ads:** Google AdSense (`ca-pub-3154853937012742`), Auto Ads on all pages

## How it works

1. **GitHub Actions** runs `daily-brief.yml` every 6 hours.
2. It runs `scripts/build_edition.py`, which:
   - Collects headlines (Google News RSS), filters to a **verified-publisher
     allowlist** (agency wires like ANI/PTI and contested outlets excluded).
   - Writes each section's stories with Claude as **editorial articles** — they
     lead with the actual news (no "according to a report…" openings), carry the
     real details (squad names, scorelines, figures), and read in proper
     paragraphs.
   - De-duplicates near-identical stories (even when headlines are reworded).
   - Sources images **Wikimedia-first**, then stock (Pexels/Unsplash/Pixabay),
     then AI illustration for concept stories, then editorial art as a last
     resort. No two stories reuse the same image in a run.
   - Tops up thin sections from recent editions so all sections stay present.
   - Generates `data.js` (homepage), per-story article pages, `archive.html`,
     `sitemap.xml`, an opinion-style `tweets/queue.json`, the Current Affairs
     page, and the FIFA World Cup 2026 page.
3. The site is static, so readers just load the published HTML/JS.

## Schedules (cron, UTC)

| Workflow | Cron | Meaning |
|---|---|---|
| `daily-brief.yml` | `15 */6 * * *` | every 6 hours |
| `social-daily.yml` | `30 2 * * *` | 08:00 IST daily |
| `buffer-publish.yml` | `30 4 * * *` | 10:00 IST daily (after Pages publishes images) |
| `weekly-newsletter.yml` | `30 2 * * 0` | Sundays 08:00 IST |

> GitHub free-tier cron can be delayed or skipped under load; scheduled runs
> only fire from the default branch. Buffer runs **later** than the social build
> on purpose, so the carousel/tweet images are live on the site before Buffer
> fetches them over the public web.

## Required configuration

**Repository Variables** (Settings → Secrets and variables → Actions → Variables):
- `SITE_URL` = `https://thelast24.in`
- `BUFFER_SCHEDULE` = `morning` (optional — spaces posts 1h apart from a morning
  start; omit for Buffer's default queue behaviour)
- `BUFFER_MORNING_START` = `8` (optional — start hour IST for morning spacing)
- `BUFFER_SPACING_MIN` = `60` (optional — minutes between morning posts)

**Repository Secrets** (Settings → Secrets and variables → Actions → Secrets):
- `ANTHROPIC_API_KEY` — required (articles, captions, current affairs, tweets)
- `PEXELS_API_KEY` — images
- `UNSPLASH_ACCESS_KEY` — images (more sources = fewer artwork fallbacks)
- `PIXABAY_API_KEY` — images
- `GEMINI_API_KEY` — optional, AI illustrations for concept stories
- `BUFFER_API_KEY` — required for any social posting via Buffer
- `BUFFER_X_CHANNEL_ID` — your X channel id in Buffer
- `BUFFER_IG_CHANNEL_ID` — your Instagram channel id in Buffer
- `BUFFER_LINKEDIN_CHANNEL_ID` — your LinkedIn channel id in Buffer

> Find Buffer channel IDs by running `python scripts/buffer_publish.py
> --list-channels` (with `BUFFER_API_KEY` set), or via Buffer's GraphQL explorer
> with `query { account { organizations { channels { id service name } } } }`.

> Without the image keys, stories fall back to branded editorial artwork — it
> won't break, you'll just see fewer photos.

**Newsletter:** the Formspree endpoint is set in `index.html`
(`NEWSLETTER_ENDPOINT = "https://formspree.io/f/mgobpqbn"`).

## Editorial principles

- **Verified publishers only.** Agency wires (ANI, PTI) and contested/low-rigor
  outlets are explicitly excluded; only reputable mastheads, quality
  international wires, and respected specialist sources are allowed.
- **Articles, not summaries.** Stories lead with the actual development (never
  "according to X reports…"), deliver the real specifics (name the squad, give
  the score and scorers, state the actual figures), and read in proper editorial
  paragraphs. Each article page shows a **Key facts** box up top.
- **Why it matters.** Every story includes a short, story-specific note on its
  relevance to an everyday Indian reader.
- **Images, responsibly.** Wikimedia real-subject photos are preferred; AI
  images are never generated of real people/events. Each story's image is
  unique within a run.

**Homepage section order:** World → Business → Tech → National → AI → Sports →
Entertainment (National is mid-page by design; the hero carousel surfaces the
most important/recent stories across all sections).

## Social

All posting goes through **Buffer** (`buffer_publish.py`), which queues content
to each channel. Buffer fetches assets over the public web, so the social build
commits images first and Buffer runs later once they're live.

- **Twitter/X — up to 10/day.** `build_edition.py` writes the day's most
  important stories to `tweets/queue.json` as **opinion-style** tweets (a sharp
  take, not a dry bulletin), with 2–3 hashtags and a link. Tweets post with an
  image when one is available, and fall back to text-only if an image can't be
  fetched (so a tweet never fails over a dead image).
- **Instagram — carousels per section.** `build_social.py` renders a cover +
  story slides + outro per section (4:5 portrait), with fuzzy cross-publisher
  de-duplication, image-bearing stories prioritised, an Instagram-native caption
  (hooks, emojis, 7–8 hashtags, save/follow CTAs), and a **PDF** of each
  carousel. Output: `social/instagram/<date>/`.
- **LinkedIn — 3/day, AI / Tech / World only.** Those sections also get a
  **separate PDF** and a **professional, opinionated caption** (strong hook,
  point of view, no external link in body, 3–5 hashtags) in
  `social/linkedin/<date>/`. LinkedIn posts the PDF as a native document
  carousel (with image fallback).

## FIFA World Cup 2026 module

`build_worldcup.py` pulls live data from **openfootball** (public domain, no API
key) and generates `worldcup.html` — group standings, latest scores, and
upcoming fixtures, all in IST. It also injects a daily **Sports carousel story**
with a full match report: each match in its own paragraph, with scorers and
minutes, half-time score, group, and venue. The story refreshes every run with
the latest pull. Toggle off after the tournament with `WORLDCUP_ENABLED`.

---

## Current Affairs (UPSC / exam-relevant)

`build_current_affairs.py` re-curates the same headlines through an exam lens,
organised into categories aspirants study by (Polity, Economy, IR, Environment,
Sci-Tech, Schemes, Defence, Reports & Indices, Persons, Places, Culture). Each
item carries a summary, the exam angle, the GS paper, and key facts. Linked
prominently from the homepage. Refreshes every 6 hours with the main run, with
a 3-day rolling retention.

## Deployment notes

- GitHub Pages serves the repo root; custom domain `thelast24.in` is set in
  Settings → Pages with HTTPS enforced. DNS (A records + `www` CNAME) is at the
  domain registrar; email MX records are left untouched.
- The service worker (`sw.js`) caches aggressively — after any change, do a
  **hard refresh (Ctrl/Cmd+Shift+R)** or use a private window to see updates.
- **Buffer posts over the public web**, not from the repo — a file being
  committed isn't enough; GitHub Pages must publish it to the live site first
  (takes a few minutes). This is why the Buffer workflow runs after the social
  build.
- Whenever the pipeline generates a file, it must be in the workflow's
  `git add` line or it won't publish.
